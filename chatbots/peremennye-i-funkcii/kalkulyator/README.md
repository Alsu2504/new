# Калькулятор

Калькулятор - это поле в Редакторе блока, которое открывается при клике по одноименной вкладке.  В этом поле можно совершать логические или арифметические действия, выполнять функции API- запросов конструктора и доступных интеграций.

В нем можно присвоить(объявить) переменную клиента, перешедшего в данный блок, или назначить значение уже существующим переменным. Например, будет полезно при создании воронки или отслеживания источника трафика.&#x20;

## **Как использовать калькулятор в конструкторе**

{% hint style="danger" %}
При обращении к переменной не нужно использовать конструкцию #{}
{% endhint %}

Переменные можно записывать просто названием или через #{}.  Все переменные автоматически будут приводиться к числам с плавающей запятой.

Калькулятор поддерживает все необходимые логические и арифметические операции.

## Оператор IF

В калькуляторе можно прописывать условия с оператором IF:

`if (score > 90) {`

&#x20;     `grade = 'отлично'`

&#x20;     `bonus = 500`

&#x20; `} else if (score > 70) {`

&#x20;     `grade = 'хорошо'`

&#x20; `} else {`

&#x20;     `grade = 'старайся лучше'`

&#x20; `}`

<div data-with-frame="true"><figure><img src="../../../.gitbook/assets/IMAGE 2026-07-15 155520.jpg" alt="" width="563"><figcaption></figcaption></figure></div>

1. Условия можно прописывать в одну строку: `if (paid == 1) { status = 'ok' } else { status = 'wait' }`
2. Внутри веток — сколько угодно строк, вложенные условия и циклы.
3. Выполняется только та ветка, чьё условие сработало. Остальные не вычисляются вообще: несуществующая переменная или ошибка в «чужой» строке больше ничего не ломает.
4. Подстановка #{переменных} тоже происходит только в активной ветке.

[Метод if(условие, если\_да, если\_нет)](uslovnyi-operator-if.md) работает как раньше, но теперь вычисляется только нужная ветка: status = if(paid == 1, 'Оплачено', 'Ждём оплату: #{payment\_url}')

Если paid == 1, ветка с #{payment\_url} даже не подставляется — переменной может не существовать.

## Циклы while и for, break и continue

`i = 0`\
`sum = 0`\
`while (i < 10) {sum += i`

&#x20;     `i += 1`

&#x20; `}`

&#x20; `for (i = 0; i < 10; i += 1) {`

&#x20;     `sum += i`

&#x20; `}`

Циклы и условия можно свободно вкладывать друг в друга.

1. break — досрочный выход из цикла
2. continue — переход к следующей итерации:

`found = -1`

&#x20; `for (i = 0; i < arr_len(items); i += 1) {`

&#x20;     `if (items[i] == target) {`

&#x20;         `found = i`

&#x20;         `break`

&#x20;     `}`

&#x20; `}`

{% hint style="info" %}
for (;;) { ... } — бесконечный цикл (до лимита).

Защита от зацикливания: на один блок переменных — суммарно не более 256 итераций всех циклов. При достижении лимита циклы останавливаются, блок продолжает работать дальше. **Досрочный break экономит этот лимит.**
{% endhint %}

### Как сохраняются переменные

1. Внутри цикла значения обновляются мгновенно: условие цикла и каждая следующая строка видят актуальные данные.
2. Сохраняются только финальные значения — один раз, после завершения цикла (в том числе через break). Независимо от количества итераций, запишется только итог.
3. Если внутри цикла произошла ошибка, то изменения из цикла не сохраняются вовсе. Значения итераций временные, сохранённым останется то, что было до входа в цикл.
4. Переменная цикла for (например, i) — служебная. Живёт только внутри цикла, никогда не сохраняется и исчезает после его завершения.

### Локальные переменные

Ключевое слово local создаёт служебную переменную: она работает во всех расчётах до конца блока, но никогда не сохраняется в карточку клиента, поэтому не нужно потом чистить временные tmp и buf через del.

&#x20; `local tmp = get(#{data}, 'items')`

&#x20; `local count = arr_len(tmp)`

&#x20; `if (count > 5) {`

&#x20;     `result = 'много'`

&#x20; `}`

Если у клиента есть обычная переменная с тем же именем — она останется нетронутой: local просто временно её заслонит.

3. greeting **`+=`** ', привет!'

## Переносы строк и JSON без кавычек

Длинное выражение больше не обязано помещаться в одну строку — раньше перенос ломал блок, что являлось частой ошибкой при работе с калькулятором.&#x20;

Теперь вызов функции можно переносить — просто разрывайте строку после запятой между аргументами:

<div data-with-frame="true"><figure><img src="../../../.gitbook/assets/Снимок экрана 2026-07-15 в 16.11.59.png" alt=""><figcaption></figcaption></figure></div>

&#x20;Словари и массивы пишутся без обёртки в кавычки и на сколько угодно строк:

&#x20; `cfg = {`

&#x20;     `"greeting": "Здравствуйте!",`

&#x20;     `"work_hours": [10, 19]`

&#x20; `}`

&#x20; `hello = cfg["greeting"]`

<div data-with-frame="true"><figure><img src="../../../.gitbook/assets/Снимок экрана 2026-07-15 в 16.13.55.png" alt=""><figcaption></figcaption></figure></div>

Единственное ограничение: внутри текста в кавычках ('...') разрывать строку нельзя. Переносите после запятой между аргументами или внутри {фигурных} и \[квадратных] скобок — там можно разрывать где угодно.
